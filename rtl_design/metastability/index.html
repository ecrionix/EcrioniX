Metastability in RTL Design

Understanding one of the most critical challenges in Clock Domain Crossing (CDC)

What is Metastability?

Metastability is a phenomenon in digital circuits where a flip-flop or latch enters an unstable, intermediate state when its setup or hold time requirements are violated. Instead of cleanly resolving to logic '0' or '1', the output can hover at an intermediate voltage level for an unpredictable amount of time before eventually settling to a stable value.

Why Does Metastability Happen?

When an asynchronous signal is sampled by a flip-flop clocked by a different clock domain, there is a chance that the data transition happens exactly during the setup or hold time window of the receiving flip-flop. This causes the flip-flop's internal feedback loop to become balanced in an unstable equilibrium.

The Solution: 2-Flip-Flop Synchronizer

module two_ff_synchronizer (
    input  wire clk_dest,
    input  wire rst_n,
    input  wire async_in,
    output reg  sync_out
);

    reg sync_ff1;

    always @(posedge clk_dest or negedge rst_n) begin
        if (!rst_n) begin
            {sync_out, sync_ff1} <= 2'b00;
        end else begin
            {sync_out, sync_ff1} <= {sync_ff1, async_in};
        end
    end

endmodule

MTBF Calculation

The reliability of a synchronizer is measured by its Mean Time Between Failures (MTBF).

MTBF Formula

MTBF = e^(t/τ) / (f_clk × f_data × T₀)

Best Practices to Handle Metastability

  • Always use at least 2-stage synchronizers for single-bit signals.
  • Use Gray coding for multi-bit buses.
  • Never synchronize multi-bit data directly.
  • Run static CDC analysis tools (SpyGlass, Cadence, etc.).
