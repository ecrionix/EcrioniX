Clock Domain Crossing (CDC) Techniques

RTL Design Best Practices for Reliable VLSI/ASIC Designs

← Back to VLSI

Clock Domain Crossing (CDC) occurs when a signal travels from one clock domain to another asynchronous clock domain. Without proper handling, it leads to metastability, data corruption, or functional failures.

Why CDC is Critical?

  • Metastability: Flip-flop setup/hold violations
  • Data Loss/Corruption: Multi-bit signals skew
  • Reduced MTBF: Poor reliability in SoCs

1. Single-Bit Synchronizers

Two-Flip-Flop Synchronizer

module two_ff_sync (
    input  wire clk_dest,
    input  wire rst_dest_n,
    input  wire async_in,
    output reg  sync_out
);
    reg sync_ff1;
    always @(posedge clk_dest or negedge rst_dest_n) begin
        if (!rst_dest_n) begin
            {sync_out, sync_ff1} <= 2'b00;
        end else begin
            {sync_out, sync_ff1} <= {sync_ff1, async_in};
        end
    end
endmodule

2. Multi-Bit Data Transfer

  • Gray Code Synchronization – Only one bit changes
  • Handshake Protocol – REQ/ACK
  • Asynchronous FIFO – Best for high bandwidth

Best Practices Checklist

  • ✓ Always use proper synchronizers
  • ✓ Gray coding for address/data buses
  • ✓ Static CDC checks (SpyGlass, etc.)
