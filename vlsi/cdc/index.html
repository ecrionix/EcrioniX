Clock Domain Crossing (CDC) Techniques

RTL Design Best Practices for Reliable VLSI/ASIC

Clock Domain Crossing (CDC) occurs when signals move between asynchronous clock domains. Proper CDC handling prevents metastability, data corruption, and functional bugs.

Why CDC Matters

  • Metastability in flip-flops
  • Multi-bit data skew
  • Reduced MTBF in SoCs

1. Single-Bit Synchronizers

module two_ff_sync (
    input wire clk_dest, rst_dest_n,
    input wire async_in,
    output reg sync_out
);
    reg sync_ff1;
    always @(posedge clk_dest or negedge rst_dest_n) begin
        if (!rst_dest_n) {sync_out, sync_ff1} <= 0;
        else {sync_out, sync_ff1} <= {sync_ff1, async_in};
    end
endmodule

2. Multi-Bit Transfer

Use Gray coding, Handshake, or Async FIFO.

Best Practices Checklist

  • Always use proper synchronizers
  • Gray code for buses
  • Static CDC checks with tools
