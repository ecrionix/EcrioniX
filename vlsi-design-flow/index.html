FRONT-END

The full RTL-to-GDSII flow, explained

Turning a chip idea into silicon is an assembly line of its own. The VLSI design flow (often summarised as RTL-to-GDSII) splits into a front-end (logical) half — define what the chip does — and a back-end (physical) half — turn that logic into a manufacturable layout. Here's each stage in depth.

1. Specification

Everything starts with the spec: the functional requirements, target performance, power and area (PPA) budgets, interfaces and process node. A weak spec poisons everything downstream, so this stage defines the architecture and the success criteria before a line of code is written.

2. RTL Design

Engineers describe the hardware's behaviour in a hardware description language — Verilog or SystemVerilog — at the register-transfer level: registers, datapaths, FSMs and how data moves on each clock. This is the human-authored "source code" of the chip. (New to it? See our ARM course and FSM Designer.)

3. Functional Verification

Before spending millions on silicon, you must prove the RTL is correct. Verification (often 60–70% of the effort) builds testbenches — typically SystemVerilog/UVM — with constrained-random stimulus, assertions and coverage to hunt bugs. Sharpen your eye with Verilog Spot the Bug.

4. Logic Synthesis

A synthesis tool converts the RTL into a gate-level netlist — actual logic gates and flip-flops from a standard-cell library — optimised for timing, area and power against the target technology. RTL becomes hardware.

5. DFT Insertion

Design-for-test adds structures (scan chains, MBIST, JTAG) so the manufactured chip can be tested for defects. It bridges front-end and back-end, and is a guaranteed step on every production chip.

6. Floorplanning

Physical design begins. Floorplanning defines the die and core area, places large macros (memories, IP blocks) and the I/O pads, and plans power distribution. A good floorplan makes the rest of the flow easy; a bad one makes timing closure a nightmare.

7. Placement

The tool positions the thousands-to-millions of standard cells into rows in the core, minimising wire length and congestion while respecting timing. The scattered logic snaps into an organised layout.

8. Clock Tree Synthesis (CTS)

The clock must reach every flip-flop at almost the same instant (low skew). CTS builds a balanced distribution network — often an H-tree with buffers — to deliver the clock cleanly across the whole die.

9. Routing

Now connect everything: routing lays down the metal wires across many layers to wire up every net — signal, clock and power — without shorts or design-rule violations. The abstract netlist becomes real interconnect.

10. Signoff & GDSII

Final checks: static timing analysis (timing closes across all corners), power analysis, and physical verification (DRC/LVS) to confirm the layout is manufacturable and matches the schematic. Pass them all and the design is exported as a GDSII file and taped out to the foundry. (Explore each career here: VLSI Career Roadmap.)

FAQ

What is the VLSI design flow?

The steps that turn a chip idea into a manufacturable layout: spec → RTL → verification → synthesis → DFT → floorplan → placement → CTS → routing → signoff/GDSII.

What is RTL-to-GDSII?

The journey from a register-transfer-level description (Verilog) to the GDSII layout file the foundry uses to make masks — via synthesis and physical implementation.

Front-end vs back-end?

Front-end = spec, RTL, verification, synthesis (the logic). Back-end = floorplanning, placement, CTS, routing, signoff (the physical layout).
