Die Planning & Floorplanning

Complete guide to die planning and floorplanning. Aspect ratio, core utilization, macro placement, power distribution, and production optimization for VLSI chips.

By EcrioniX · Published June 14, 2026 · ~4500 words · 14 min read

1. What is Die Planning?

Die planning is the first critical step in physical design. It determines the chip's overall dimensions, aspect ratio, placement zones, and power distribution strategy before detailed layout.

Key decisions made during die planning:

  • Die size and aspect ratio (square vs rectangular)
  • Macro placement (memories, hardmacros)
  • Core area utilization target (70-80% typical)
  • Power delivery network (PDN) strategy
  • I/O ring and pad placement
  • Routing resources allocation
  • Clock distribution regions

Impact: Poor die planning forces expensive redesigns later. Good planning ensures timing closure, power efficiency, and manufacturability.

2. Aspect Ratio Selection

Aspect Ratio = Die Width / Die Height

Common aspect ratios: 1.0 (square): Most balanced, minimal perimeter 1.5 (slightly wider): Easier to fit logic, better for certain macro placement 2.0 (wide): Helps with horizontal routing, but increases routing congestion 0.67 (taller): Vertical-friendly, harder to route Example: 100 mm² die Aspect 1.0 (square): 10mm × 10mm perimeter = 40mm Aspect 1.5 (wide): 12.2mm × 8.2mm perimeter = 40.8mm

Visual comparison of die aspect ratios:

3. Core Utilization Target

Core utilization = (Standard cell area + Macro area) / Available core area

Typical targets by technology:

Technology NodeTarget UtilizationRouting CongestionTiming Risk
28nm+60-65%LowLow
14nm70-75%ModerateModerate
7nm75-80%HighHigh
5nm80-85%Very HighVery High

Why not 100% utilization?

  • Routing needs space (metal layers, vias)
  • Timing optimization needs whitespace (cell movement)
  • Power distribution needs tracks (metal resources)
  • Higher utilization → exponentially harder to close timing

4. Macro Placement Strategy

Macros are pre-designed blocks: memories (SRAM), hardmacros (PLL, analog), embedded processors

Floorplan with Macro Placement: ┌─────────────────────────────────────┐ │ I/O Ring (Pads) │ ├──────────────────────────────────────┤ │ ┌──────────┐ ┌──────────┐ │ │ │ SRAM │ │ SRAM │ │ │ │ 64KB │ │ 64KB │ │ │ └──────────┘ └──────────┘ │ │ │ │ ┌────────────────────────────────┐ │ │ │ Core Logic (Standard Cells) │ │ │ │ Utilization: 75% │ │ │ └────────────────────────────────┘ │ │ │ │ ┌──────────┐ ┌──────────┐ │ │ │ Hardmacro│ │ Hardmacro│ │ │ │ (PLL) │ │ (PHY) │ │ │ └──────────┘ └──────────┘ │ └──────────────────────────────────────┘ Placement Principles: 1. Co-locate SRAMs near logic that uses them (minimize wire delay) 2. Group memories hierarchically (L1 near CPU, L2/L3 near periphery) 3. Place power-hungry macros symmetrically (even heat distribution) 4. Route data busses directly from macro to core (minimize hops)

5. Power Delivery Network (PDN) Planning

PDN delivers power from package pads to every transistor. Poor PDN causes:

  • Voltage droop (IR drop)
  • Timing violations (slower gates at lower voltage)
  • Functional failures (logic at wrong voltage)
  • Electromigration (metal failure from high current density)

PDN structure (hierarchical):

  1. Package (coarsest): Large capacitors on PCB
  2. Die (medium): Decoupling capacitors (on-die cap)
  3. Cell (finest): Local power straps (metal routing)
PDN Hierarchy (VDD Distribution): ┌─ Package Bulk Cap (100µF) PCB │ ┌──────────────┴──────────────┐ │ │ VDD_PAD VDD_PAD │ 1mΩ │ 1mΩ └────────┬─────────────────┬─┘ │ Decap Array │ ┌────┴────────┬─────────┴──┐ │ Die Caps │ │ 10µF 10µF 10µF │ │ │ ┌────┴──────────────┴────────────┴────┐ │ │ M9 Power Straps (1µm wide, every 20µm) │ ├─ M7 Power Rails │ ├─ M5 Power Rails │ ├─ M3 Local PDN (80% area) │ │ │ ├─ Standard Cells (VDD/GND per cell) │ │ │ └───────────────────────────────────────┘
PDN Impedance Budget: Max allowable voltage drop = 10% of supply (1.8V supply → max 180mV drop) PDN impedance targets: Package: 1-2 mΩ (bulk power) Die: 2-5 mΩ (distribution) Cell: 1-3 mΩ (local) Total: < 10mΩ (ensures < 180mV @ 18A peak) Design Rule: Power grid straps every 100-200 µm (tight spacing) Ground return path in parallel (minimize loop inductance)

6. I/O Ring & Pad Placement

I/O ring: Physical boundary with pads (power, ground, data, control)

Considerations:

  • Pad ring area: ~1-2% of die area (hundreds of pads)
  • Power pads: Frequent spacing (every 4-8 data pads)
  • Ground pads: Equal or greater than power (return path)
  • Data pads: Grouped by function (control, address, data)
  • Bump placement: For BGA packages (ball grid array)

Example (256-pin package):

  • 50 power pads (Vdd)
  • 50 ground pads (Gnd)
  • 100 signal pads (I/O)
  • 56 spare pads (future use or redundancy)

7. Core Region Planning

Core region: Area available for standard cells (logic)

Typical allocation in 100 mm² die:

  • Macros (SRAM, hardmacro): 20-30 mm²
  • Core logic: 50-70 mm²
  • Routing overhead: 5-10 mm² (metal resources)
  • Power/ground straps: 5-10 mm² (metal area)
  • I/O ring: 5-10 mm²

8. Real-World Floorplan Examples

Mobile Processor (Apple A17)

  • Die size: ~150 mm² (5nm)
  • 6 CPU cores + GPU + Neural Engine
  • Unified memory hierarchy (L1 → L2 → L3)
  • High core utilization (80%+) → aggressive power delivery
  • Aspect ratio: ~1.2 (slightly wider for pipeline routing)

Server Processor (AMD EPYC)

  • Die size: ~80 mm² per chiplet (7nm)
  • Modular design: 8-12 chiplets connected via Infinity Fabric
  • Each chiplet has its own floorplan with macros
  • Central controller chiplet (smaller, 30 mm²)
  • Aspect ratio: Optimized for chiplet connectivity

9. Design Checklist: Die Planning

  • ✅ Calculate target die size: Based on logic gates, memory, I/O count
  • ✅ Choose aspect ratio: 1.0 (square) for most, 1.5-2.0 if macro alignment critical
  • ✅ Set core utilization: 70-80% for advanced nodes, 60-65% for older
  • ✅ Place macros: Group by function, minimize wire delay
  • ✅ Plan PDN: Power straps every 100-200 µm, decap planning
  • ✅ Design I/O ring: Power pads frequent, ground equal or greater
  • ✅ Allocate routing resources: 20-30% of metal for power/clock
  • ✅ Plan clock regions: Separate clock trees per major block
  • ✅ Document floorplan: Ownership, constraints, design rules
  • ✅ Review manufacturability: DFM rules, litho constraints
