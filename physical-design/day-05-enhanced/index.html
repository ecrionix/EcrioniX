Routing & Timing Closure

Routing strategies, metal layers, via placement, timing closure techniques, DRC/LVS verification, and production optimization.

By EcrioniX · Published June 14, 2026 · ~4400 words · 13 min read

1. Metal Layers & Via Strategy

Routing uses multiple metal layers (M1-M10 in advanced nodes). Each layer has specific uses:

LayerUsePitchResistance
M1Local interconnect, cell signals20-40nmHigh (thin)
M2-M6Local/regional routing40-100nmMedium
M7-M9Global routing, power distribution200-500nmLow (thick)
M10Top metal, bumps500-1000nmVery low

Via strategy: Use larger vias on lower layers (less area), smaller vias on higher layers.

2. Global vs Detailed Routing

Global Routing

Coarse routing determining layer and general path.

  • Uses abstracted grid (represents multiple metal layers)
  • Determines if routing is feasible without congestion
  • Time: Seconds to minutes

Detailed Routing

Fine-grained routing on actual metal layers.

  • Places metal and vias to avoid DRC violations
  • Meets timing by sizing wires and placing buffers
  • Time: Hours to days (depends on size)

3. Timing Closure Techniques

Timing closure ensures all paths meet setup/hold times.

Timing equation: Setup check: data_arrival + clock_skew < clock_period - setup_time If violated: insert buffer on data path or reduce clock skew Hold check: data_arrival - clock_skew > hold_time If violated: insert delay element or increase clock skew Common fixes (in order of preference): 1. Buffer insertion (increases delay on slack paths) 2. Upsizing cells (reduces delay on critical paths) 3. Gate duplication (parallelize logic) 4. Routing optimization (reduce wire delays) 5. Clock skew adjustment (careful, affects hold time)

4. Congestion-Driven Routing

Iterative refinement to resolve routing bottlenecks.

  • Ripup and reroute: Remove routes in congested areas and reroute through less congested paths
  • Placement adjustment: Move cells to reduce demand on congested areas
  • Buffer insertion: Add buffers in uncongested areas (uses whitespace)

5. Design Rule Checks (DRC)

DRC ensures routing meets manufacturing design rules.

RuleTypical Value (5nm)Why It Matters
Minimum spacing20-30nmLithography resolution
Minimum line width20nmMinimum feature size
Via array rulesVia spacingResistance, yield
Metal density80-90%CMP polishing uniformity
Antenna rulesGate oxide limitsGate damage during implant

DRC violations can be caught early and fixed before tape-out.

6. LVS (Layout vs Schematic) Verification

Ensures layout matches netlist (logic equivalence).

  • Extracts connectivity from layout (metal/via connectivity)
  • Compares against netlist (what logic was supposed to do)
  • Reports shorts, opens, and missing connections

7. Real-World Routing Examples

7nm Mobile Chip

  • 100M nets (signal connections)
  • Global routing: 1 hour
  • Detailed routing: 24 hours
  • DRC iterations: 3-5 (typical)
  • Timing closure: 98% paths met

5nm Processor

  • 1B+ nets (massive routing problem)
  • Hierarchical routing (route by sub-block)
  • Timing closure harder (tight skew budgets)
  • Multiple DRC iterations (tight design rules)

8. Routing Checklist

  • ✅ Global routing feasibility: Ensure no routing impossible regions
  • ✅ Layer assignment: Use appropriate metal layers by signal importance
  • ✅ Via strategy: Minimize area while ensuring reliability
  • ✅ Timing closure: Meet setup/hold with buffer insertion
  • ✅ Congestion resolution: Ripup/reroute iteratively
  • ✅ DRC verification: All spacing and density rules met
  • ✅ LVS verification: Layout matches netlist (electrical connectivity)
  • ✅ Signal integrity: Check crosstalk and slew violations
  • ✅ Power/ground distribution: Verify PDN adequacy
  • ✅ Final sign-off: Ready for tape-out
